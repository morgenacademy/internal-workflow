Wat ga je doen?

Kies wat van toepassing is. Zo lees je alleen wat nu relevant is.

1. Nieuw project Ik begin met iets nieuws dat nog niet bestaat. 2. Bestaand project Ik werk verder aan een project dat al bestaat. 3. Klaar & opslaan Ik heb gewerkt en wil het bewaren of online zetten. 4. Status checken Ik wil weten hoe het ervoor staat, of ik zit vast.

1. Ik begin een nieuw project

Je maakt het project in één keer aan op GitHub én op je computer.

Hoe werkt dit? We laten GitHub de baas zijn. Met één commando maak je de repo op GitHub aan en zet je 'm meteen als kloon in je Repos-map. De lokale map en GitHub zijn dan al aan elkaar gekoppeld — je hoeft achteraf niets meer te verbinden.
1
Open Finder en ga naar Documenten → Repos.
2
Rechtsklik op de map Repos en kies Nieuwe Terminal bij map (of: New Terminal at Folder).
3
Kopieer het commando hieronder, vervang projectnaam door een logische naam, plak het in Terminal en druk op Enter.
1
Open Verkenner en ga naar Documenten → Repos.
2
Klik in de adresbalk van Verkenner, typ powershell en druk op Enter. PowerShell opent nu in de map Repos.
3
Kopieer het commando hieronder, vervang projectnaam door een logische naam, plak het in PowerShell en druk op Enter.
Repo aanmaken én ophalen 
gh repo create projectnaam --private --clone
Wat doet dit?
  • gh repo create maakt de repo aan op GitHub.
  • --private houdt het project privé. Wil je het openbaar? Gebruik dan --public.
  • --clone zet de map meteen op je computer, in je Repos-map.
En daarna? Je hebt nu een map projectnaam in je Repos-map. Open die map met Claude, Codex, Antigravity of Conductor en ga aan de slag. Klaar met werken? Ga naar 3. Klaar & opslaan.

2. Ik werk aan een bestaand project

Het project bestaat al op GitHub. Eerst de vraag: staat het ook al op jóuw computer?

Situatie A — ik heb het project nog niet op mijn computer

Bijvoorbeeld: een collega is ermee begonnen, of je werkt op een nieuwe computer. Dan haal je het project één keer op. Dat heet clonen.

1
Open het project op github.com in je browser.
2
Klik op de groene knop Code en kopieer de HTTPS-link.
3
Open Terminal in je Repos-map (rechtsklik op Documenten → Repos → Nieuwe Terminal bij mapin Verkenner naar Documenten → Repos, typ powershell in de adresbalk).
4
Kopieer het commando hieronder, vervang REPO-URL door de gekopieerde link en druk op Enter.
Project ophalen van GitHub 
git clone REPO-URL

Situatie B — ik heb het project al op mijn computer

Begin altijd met git pull, ook als je denkt dat je de nieuwste versie hebt. Je kunt het nooit zeker weten — een collega kan ondertussen iets gewijzigd hebben.

1
Open Finder, ga naar Documenten → Repos → je projectmap.
2
Rechtsklik op de projectmap en kies Nieuwe Terminal bij map.
3
Typ git pull en druk op Enter. Daarmee haal je de nieuwste versie binnen.
1
Open Verkenner, ga naar Documenten → Repos → je projectmap.
2
Klik in de adresbalk, typ powershell en druk op Enter.
3
Typ git pull en druk op Enter. Daarmee haal je de nieuwste versie binnen.
Nieuwste versie ophalen 
git pull
Is git pull veilig? Ja, altijd. git pull haalt de nieuwste versie van GitHub op én voegt die samen met wat jij lokaal hebt. Het overschrijft jouw werk nooit: heb je nog niet-opgeslagen of niet-gepushte wijzigingen, dan houdt Git die. Daarom: gewoon altijd git pull voordat je begint. Daarna open je het project met Claude, Codex, Antigravity of Conductor.

3. Ik ben klaar en wil opslaan

Twee dingen: je werk veilig terugzetten naar GitHub, en eventueel je project online zetten.

Stap A — Je werk opslaan naar GitHub

Waar typ ik dit? Niet in Claude of Antigravity zelf, maar in Terminal, geopend in dezelfde projectmap waarin je net gewerkt hebt. Open Terminal via Finder → rechtsklik op de projectmap → Nieuwe Terminal bij map.
Waar typ ik dit? Niet in Claude of Antigravity zelf, maar in PowerShell, geopend in dezelfde projectmap waarin je net gewerkt hebt. Open PowerShell via Verkenner → adresbalk → powershell.
Korte versie (na de setup-shortcut) 
save
Of de drie losse commando's 
git add .
git commit -m "update"
git push
Wat doet dit?
  • git add . — verzamelt al je wijzigingen.
  • git commit -m "update" — slaat ze op met een korte omschrijving.
  • git push — stuurt ze naar GitHub.

Stap B — Je project online zetten (deploy)

Waarom deployen? Zolang je project alleen op GitHub staat, kan niemand het als website bezoeken. Door het te deployen via Netlify wordt het automatisch online gezet op een publieke URL. Elke keer dat je iets naar GitHub pusht, wordt de website automatisch bijgewerkt. Dit hoef je maar één keer per project in te stellen.
1
Ga naar app.netlify.com en log in met je GitHub-account (of maak een account aan).
2
Klik op Add new project.
3
Kies Import an existing project en selecteer GitHub als bron.
4
Kies de repo van jouw project uit de lijst. Zie je hem niet? Geef Netlify dan eerst toegang tot je GitHub-account.
5
Vul de build settings in. Weet je niet wat hier moet? Vraag het aan de AI-tool waar je mee bouwt (Claude, Codex, Antigravity).
6
Optioneel: vul environment variables in als jouw project die nodig heeft. Ook hier: vraag het aan je AI-tool.
En daarna? Je krijgt een URL van Netlify (iets als jouw-project.netlify.app). Wil je een eigen domeinnaam zoals jouwproject.nl? Dat stel je in onder Domain settings in Netlify.

4. Status checken / vastgelopen?

Een foutmelding, of je weet niet meer wat de nieuwste versie is? Begin hier.

Waar typ je dit? Open Finder, ga naar Documenten → Repos → je projectmap. Rechtsklik op de map en kies Nieuwe Terminal bij map.
Waar typ je dit? Open Verkenner, ga naar Documenten → Repos → je projectmap. Klik in de adresbalk, typ powershell en druk op Enter.
Bekijk de stand van zaken 
git fetch
git status
Belangrijk: git status toont vaak meerdere regels tegelijk. Loop ze allemaal na. Zoek elke regel van je scherm op in de tabel hieronder. Staat er bij een regel een actie? Voer die uit. Zie je alleen working tree clean én up to date en verder niets? Dan ben je klaar.
Wat er op je scherm staat Wat dat betekent Wat doe je?
nothing to commit, working tree clean Je hebt geen losse wijzigingen — alles wat je hebt gedaan is opgeslagen. Niets doen. Check ook de regel over je branch.
modified: ... Je hebt dit bestand aangepast, maar die wijziging is nog niet opgeslagen. Typ save.
Untracked files Een nieuw bestand dat Git nog niet kent en dat nog niet is opgeslagen. Typ save — neemt het meteen mee.
Your branch is up to date Je opgeslagen versie is gelijk aan die op GitHub. Niets doen. Check ook of er modified-regels staan.
Your branch is ahead Je hebt opgeslagen werk dat nog niet op GitHub staat. Typ save.
Your branch is behind Een collega heeft iets gewijzigd dat jij nog niet hebt. Typ git pull.
Fout: "not a git repository"? Dan zit je niet in een projectmap. Open Terminal opnieuw, maar nu vanuit de juiste map in Documenten → Repos: rechtermuisknop op de map → Nieuwe Terminal bij map (of: New Terminal at Folder).
Fout: "not a git repository"? Dan zit je niet in een projectmap. Open PowerShell opnieuw, maar nu vanuit de juiste map in Documenten → Repos: klik in de adresbalk van Verkenner, typ powershell en druk op Enter.
Kom je er niet uit? Zie je een andere foutmelding, of lukt het niet? Kopieer de volledige tekst uit Terminal en plak die in de AI-tool waar je mee werkt (Claude, Codex, Antigravity) met de vraag wat er aan de hand is. Of vraag het een collega. Doe in de tussentijd niets wat je niet zeker weet — dan kan er ook niets misgaan.

Eenmalige setup

Dit doe je één keer per computer. Werk je later op een andere computer, dan doorloop je deze stappen daar opnieuw. Daarna hoef je dit nooit meer aan te raken.

1. Open Terminal

Druk ⌘ + spatie, typ Terminal en druk op Enter. Alle commando's hieronder plak je hierin, telkens gevolgd door Enter.

2. Installeer Homebrew

Wat is dit? Homebrew is een pakketbeheerder voor de Mac: een programma waarmee je andere programma's netjes installeert. We gebruiken het om Git en de GitHub CLI te installeren.
Homebrew installeren 
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Volg de instructies op het scherm. Aan het einde toont Homebrew soms twee regels die je nog moet plakken (beginnend met echo en eval) — doe dat als daarom gevraagd wordt.

3. Installeer Git en de GitHub CLI

Wat zijn dit? Git houdt versies van je projecten bij. De GitHub CLI (gh) regelt je verbinding met GitHub — daarmee maak je later projecten aan en log je veilig in.
Git + GitHub CLI installeren 
brew install git gh

4. Log in bij GitHub

Zo weet GitHub dat jij het bent. Hierna werken push en pull zonder telkens om een wachtwoord te vragen.

Inloggen bij GitHub 
gh auth login
Welke antwoorden kies je? Terminal stelt een paar vragen. Kies: GitHub.com → HTTPS → bij "authenticate Git with your GitHub credentials" antwoord Yes → Login with a web browser. Je browser opent — log in en je bent klaar.
Welk account? Bij Morgen Academy log je in met het gedeelde GitHub-account morgenacademy. Zo komt al het werk op de juiste plek terecht.

5. Vertel Git wie je bent

Vervang de naam en het e-mailadres door die van jou. Dit komt bij je opgeslagen werk te staan.

Je naam en e-mail instellen 
git config --global user.name "Jouw Naam"
git config --global user.email "jouwmail@voorbeeld.nl"

6. Maak je Repos-map aan

Hier komen voortaan al je projecten in te staan, zodat je altijd weet waar ze zijn.

Repos-map aanmaken 
mkdir -p ~/Documents/Repos

7. Stel de save-shortcut in (optioneel)

Waarom zou je dit doen? Hierna kun je in Terminal gewoon save typen om je werk op te slaan, in plaats van elke keer drie losse commando's.
1
Open het instellingenbestand met het commando hieronder. Er opent een teksteditor in Terminal.
Open het instellingenbestand 
nano ~/.zshrc
2
Je ziet een zwart scherm met bovenaan GNU nano. Plak de regel hieronder erin.
Plak deze regel in nano 
alias save="git add . && git commit -m 'update' && git push"
3
Druk op Control + O (de letter O) en daarna Enter om op te slaan.
4
Druk op Control + X om nano af te sluiten.
5
Activeer de shortcut met het commando hieronder.
Shortcut activeren 
source ~/.zshrc
Klaar! Je computer is nu volledig ingesteld. Ga naar het tabblad Dagelijks werken om aan de slag te gaan.

1. Open PowerShell

Klik op de Windows-startknop, typ PowerShell en klik erop. Alle commando's hieronder plak je hierin, telkens gevolgd door Enter.

2. Installeer Git en de GitHub CLI

Wat zijn dit? Git houdt versies van je projecten bij. De GitHub CLI (gh) regelt je verbinding met GitHub. We installeren ze met winget, de pakketbeheerder die standaard op Windows aanwezig is.
Git + GitHub CLI installeren 
winget install --id Git.Git -e
winget install --id GitHub.cli -e

Werkt winget niet? Dan kun je Git ook installeren via git-scm.com en de GitHub CLI via cli.github.com. Sluit daarna PowerShell en open het opnieuw, zodat de nieuwe programma's herkend worden.

3. Log in bij GitHub

Zo weet GitHub dat jij het bent. Hierna werken push en pull zonder telkens om een wachtwoord te vragen.

Inloggen bij GitHub 
gh auth login
Welke antwoorden kies je? PowerShell stelt een paar vragen. Kies: GitHub.com → HTTPS → bij "authenticate Git with your GitHub credentials" antwoord Yes → Login with a web browser. Je browser opent — log in en je bent klaar.
Welk account? Bij Morgen Academy log je in met het gedeelde GitHub-account morgenacademy. Zo komt al het werk op de juiste plek terecht.

4. Vertel Git wie je bent

Vervang de naam en het e-mailadres door die van jou. Dit komt bij je opgeslagen werk te staan.

Je naam en e-mail instellen 
git config --global user.name "Jouw Naam"
git config --global user.email "jouwmail@voorbeeld.nl"

5. Maak je Repos-map aan

Hier komen voortaan al je projecten in te staan, zodat je altijd weet waar ze zijn.

Repos-map aanmaken 
mkdir $env:USERPROFILE\Documents\Repos
Klaar! Je computer is nu volledig ingesteld. Ga naar het tabblad Dagelijks werken om aan de slag te gaan.
