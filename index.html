Morgen Academy – Git workflow

Deze uitleg is bedoeld voor mensen die niet dagelijks met Git of Terminal werken. Daarom staat hieronder niet alleen wat je moet doen, maar ook waarom, waar en wanneer.

Eerst het idee in gewone taal

Git is eigenlijk alleen een manier om één gedeelde bron van waarheid te hebben voor jullie projectbestanden.

Wat is hier het project?

Een project is gewoon een map op je computer waar de website of tool in staat. Bijvoorbeeld website of internal-workflow. In die map staan bestanden zoals index.html, afbeeldingen, styles en code.

Wat is de hoofdregel?

Je gebruikt altijd dezelfde projectmap, ook als je wisselt tussen Claude, Codex, Conductor of Antigravity. De tool mag wisselen. De map niet.

Belangrijk Je hoeft dus niet opnieuw te klonen als je van tool wisselt. Je opent gewoon weer dezelfde map en haalt eerst de nieuwste versie op.

0. Eenmalig instellen

Dit doe je één keer per project. Daarna wordt werken veel makkelijker.

Waarom doen we dit?

Waarom niet gewoon bestanden heen en weer sturen? Omdat GitHub dan bijhoudt wat de laatste versie is, wie iets aangepast heeft en hoe iedereen op dezelfde basis verder werkt. Dat voorkomt losse kopieën en verwarring.

Waar zet ik mijn projectmap neer?

Maakt die locatie uit? Niet heel veel, zolang het maar een vaste logische plek op je eigen computer is. Daarom spreken we af dat projecten in één map komen: ~/Repos C:\Repos . Dan weet iedereen waar projecten staan en kun je ze makkelijk terugvinden.
1
Installeer Git op je computer via git-scm.com.
2
Maak één vaste map voor projecten: ~/ReposC:\Repos.
3
Clone daarna de GitHub-repo van het project naar die map.

Wat betekent “clone repo” eigenlijk?

Gewone taal “Clonen” betekent: je haalt een kopie van de projectmap van GitHub naar je eigen computer, zodat je erin kunt werken.

Waar haal ik de repo-URL vandaan?

1
Open het project op GitHub in je browser.
2
Klik op de groene knop Code.
3
Kopieer de HTTPS-link. Dat is de URL die je hieronder op de plek van REPO-URL zet.
Clone project (Mac) 
cd ~/Repos
git clone REPO-URL projectnaam
Clone project (Windows) 
cd C:\Repos
git clone REPO-URL projectnaam
Wat moet je vervangen? Vervang REPO-URL door de link van GitHub. Vervang projectnaam door een logische mapnaam, bijvoorbeeld website of internal-workflow.

1. Save shortcut instellen

Dit is optioneel, maar wel handig. Zonder dit moet je drie losse Git-commando’s typen in plaats van één korte opdracht.

Waarom willen we dit?

Praktisch voordeel Als je de shortcut instelt, hoef je later alleen save te typen. Zonder die shortcut moet je elke keer drie regels typen. Voor beginners is één kort commando veel overzichtelijker.
Open shell-configuratie (Mac) 
nano ~/.zshrc
Windows-gebruikers 
Gebruik op Windows voorlopig liever de 3 losse Git-commando’s hieronder.
Voeg deze regel toe (Mac) 
alias save="git add . && git commit -m 'update' && git push"
Activeer de wijziging (Mac) 
source ~/.zshrc
Als je géén save gebruikt, doe je dit 
git add .
git commit -m "update"
git push

2. Wat is mijn projectmap precies?

Dit is de map waar het project zelf in staat. Dus niet zomaar een losse map, maar de echte projectmap.

Voorbeeld projectmap

~/Repos/internal-workflowC:\Repos\internal-workflow

Daarin staan bijvoorbeeld bestanden als index.html, afbeeldingen en andere projectbestanden.

Hoe herken je dat?

Dat is meestal de map die je gecloned hebt van GitHub. Als je twijfelt, open die map en kijk of daar de projectbestanden direct in staan.

3. Begin van je werkdag

Hier zit de meeste verwarring. Dus hieronder heel letterlijk wat je doet.

Wanneer typ ik git pull?

Antwoord Pas nadat je Terminal of PowerShell hebt geopend in de juiste projectmap. Dus niet zomaar in een lege Terminal op je bureaublad of home-map.

Hoe kom ik in de juiste map?

1
Open Finder.
2
Ga naar je projectmap, bijvoorbeeld Repos → internal-workflow.
3
Rechtsklik in die map en kies Open Terminal at Folder.
4
Nu staat je Terminal meteen op de juiste plek. Dán typ je git pull.
1
Open Verkenner.
2
Ga naar je projectmap, bijvoorbeeld C:\Repos\internal-workflow.
3
Klik in de adresbalk, typ powershell en druk Enter.
4
PowerShell opent nu in de juiste map. Dán typ je git pull.
Haal de nieuwste versie op 
git pull

4. Werken en daarna opslaan

Je werkt meestal in Claude Code, Codex, Antigravity of Conductor. Het opslaan doe je daarna in Terminal of PowerShell, in dezelfde projectmap.

Dus: waar typ ik save?

Antwoord Niet in Claude of Antigravity zelf, maar in Terminal of PowerShell, terwijl die open staat in dezelfde projectmap waarin je net gewerkt hebt.
1
Werk aan je project in Claude, Codex, Antigravity of Conductor.
2
Open Terminal of PowerShell in diezelfde projectmap.
3
Typ save óf de 3 losse Git-commando’s.
Korte versie met shortcut 
save
Zonder shortcut 
git add .
git commit -m "update"
git push

5. Wisselen tussen AI-tools

Dat mag. Maar je blijft wel steeds in dezelfde projectmap werken.

1
Je werkte in Claude Code aan project internal-workflow.
2
Je wilt later verder in Antigravity of Codex.
3
Open opnieuw dezelfde map internal-workflow.
4
Doe eerst git pull.
5
Werk verder en sla daarna op via Terminal of PowerShell.
