Zo werken we samen in Git zonder gedoe

Voor iedereen die met Claude Code, Codex, Conductor of Antigravity werkt: gebruik altijd één vaste lokale repo per project. Dus niet opnieuw klonen als je van tool wisselt. Start met git pull, werk in dezelfde map en sluit af met save.

1. De hoofdregel

Iedere tool werkt op dezelfde projectmap. De tool mag wisselen, de repo-map niet.

Goed om te onthouden Een GitHub-repo is jullie centrale waarheid. Jouw lokale clone is je werkmap. Claude, Codex, Conductor en Antigravity zijn alleen verschillende manieren om in diezelfde map te werken.

Niet doen Niet opnieuw klonen zodra je van tool wisselt. Niet werken vanuit een gedeelde Drive-map. Niet meerdere clones maken van hetzelfde project zonder reden.

2. Eerste keer instellen

Dit doe je één keer per project.

1
Maak op je Mac een vaste map voor projecten, bijvoorbeeld ~/Repos.
Maak op Windows een vaste map voor projecten, bijvoorbeeld C:\Repos.
2
Clone de GitHub-repo naar die map.
3
Werk voortaan altijd in diezelfde map. Dus niet later opnieuw klonen als een andere tool verder moet.
Clone een nieuwe repo (Mac)
cd ~/Repos
git clone REPO-URL projectnaam
Clone een nieuwe repo (Windows)
cd C:\Repos
git clone REPO-URL projectnaam
Voorbeeld Als je repo internal-workflow- heet, dan krijg je een map zoals ~/Repos/internal-workflow- C:\Repos\internal-workflow-.

3. Vandaag beginnen

Dit is de standaardroutine als je aan een bestaand project verdergaat.

1
Open de bestaande projectmap, bijvoorbeeld ~/Repos/website.
Open de bestaande projectmap, bijvoorbeeld C:\Repos\website.
2
Open die map in de tool waarmee je wilt werken.
3
Doe eerst git pull om de nieuwste versie van GitHub binnen te halen.
Start van je werkdag (Mac)
cd ~/Repos/website
git pull
Start van je werkdag (Windows)
cd C:\Repos\website
git pull
Waarom git pull? Dit haalt de nieuwste wijzigingen van GitHub binnen in jouw bestaande lokale clone. Dus: pull in plaats van opnieuw klonen.

4. Terminal / PowerShell openen

Open het project altijd in de juiste map, niet ergens los op je computer.

1
Open Finder en ga naar je projectmap, bijvoorbeeld Repos → website.
2
Rechtsklik in die map.
3
Kies New Terminal at Folder.
4
Je Terminal opent dan direct in de juiste repo-map.
1
Open Verkenner en ga naar je projectmap, bijvoorbeeld C:\Repos\website.
2
Klik in de adresbalk van de map.
3
Typ powershell en druk Enter.
4
PowerShell opent dan direct in de juiste repo-map.
Twijfel je of je goed zit? Gebruik git rev-parse --show-toplevel. Dan zie je direct de hoofdmap van de repo.

5. Opslaan en pushen

We gebruiken hiervoor de shortcut save.

Alles opslaan naar GitHub
save
Wat doet save? git add . + git commit -m "update" + git push
1
Werk in je projectmap.
2
Controleer eventueel met git status wat er veranderd is.
3
Typ save als je klaar bent met een logische wijziging.

6. Wisselen tussen tools

Dat is prima. Je hoeft alleen niet opnieuw te klonen.

1
Je werkte vanochtend in Claude Code in je projectmap.
2
Later zijn je tokens op en wil je verder in Antigravity, Codex of Conductor.
3
Open dan exact dezelfde map opnieuw.
4
Doe eerst git pull.
5
Werk verder en eindig weer met save.
De kern Je wisselt van tool, niet van projectmap.

7. Veelgemaakte fouten

Hier gaat het meestal mis.

Niet goed

Repo opnieuw klonen omdat je naar een andere tool overstapt.

Wel goed

Bestaande repo-map openen en eerst git pull doen.

Niet goed

Werken vanuit Drive, Dropbox of iCloud als actieve repo-map.

Wel goed

Repo lokaal in ~/Repos C:\Repos bewaren en GitHub gebruiken voor sync.

Niet goed

Geen idee hebben wat er lokaal veranderd is.

Wel goed

Eerst git status doen om overzicht te krijgen.

8. Hulp bij problemen

Met deze commando’s kom je meestal al heel ver.

Wat is er veranderd?
git status
Wat zijn de exacte verschillen?
git diff
Wat is de repo-root?
git rev-parse --show-toplevel
Ik weet niet zeker of ik in de juiste map zit

Doe git status. Krijg je een normale Git-uitvoer zoals On branch main, dan zit je waarschijnlijk goed. Met git rev-parse --show-toplevel zie je ook direct de hoofdmap van de repo.

Ik heb lokaal dingen veranderd, maar nog niet gepusht

Doe eerst git status. Gooi je map niet weg. Kijk rustig wat er veranderd is. Vaak kun je gewoon doorgaan en daarna save doen.

Ik wil vandaag alleen de nieuwste versie ophalen

Open de bestaande repo-map en gebruik git pull. Dat is meestal alles wat je nodig hebt om verder te werken.
