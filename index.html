Waarom doen we dit?

Omdat iedereen dan aan dezelfde versie werkt en niet met losse kopieën hoeft te mailen of appen.

Wat levert het op?

Je weet sneller wat de nieuwste versie is, je kunt makkelijker samenwerken en je raakt minder snel werk kwijt.

Wat is een project?

Gewoon een map op je computer met bestanden zoals index.html, afbeeldingen en scripts.

3. Ik heb een nieuw project

Gebruik deze route als het project nog niet op GitHub staat.

Wat bedoelen we hiermee? Stel: je hebt al een map op je computer met bestanden erin, maar GitHub weet nog niet dat dit project bestaat. Dan moet je die map één keer voor het eerst op GitHub zetten.
1
Maak eerst een nieuwe lege repo op GitHub.
2
Open Finder en ga naar de map waar je projectbestanden in staan, bijvoorbeeld Documenten → Repos → mijn-project.
3
Rechtsklik op de projectmap en kies Nieuwe Terminal bij map (of: New Terminal at Folder). Er opent nu een Terminal die al in de juiste map staat.
4
Kopieer het blok hieronder, plak het in Terminal, vervang REPO-URL door de link van GitHub en druk op Enter.
1
Maak eerst een nieuwe lege repo op GitHub.
2
Open Verkenner en ga naar de map waar je projectbestanden in staan, bijvoorbeeld Documenten → Repos → mijn-project.
3
Klik in de adresbalk van Verkenner, typ powershell en druk op Enter. Er opent nu een PowerShell-venster dat al in de juiste map staat.
4
Kopieer het blok hieronder, plak het in PowerShell, vervang REPO-URL door de link van GitHub en druk op Enter.
Project voor het eerst naar GitHub sturen 
git init
git add .
git commit -m "first commit"
git remote add origin REPO-URL
git push -u origin main
Wat doet dit? Hiermee maak je van jouw bestaande map een Git-projectmap en stuur je die voor het eerst naar GitHub.

Ik weet niet meer wat de laatste versie is

Gebruik dit als je twijfelt of jouw computer of GitHub de nieuwste versie heeft.

Hoe doe je dit? Open Finder, ga naar Documenten → Repos → je projectmap. Rechtsklik op de map en kies Nieuwe Terminal bij map. Typ dan de commando's hieronder.
Hoe doe je dit? Open Verkenner, ga naar Documenten → Repos → je projectmap. Klik in de adresbalk, typ powershell en druk op Enter. Typ dan de commando's hieronder.
Controleer waar de nieuwste versie staat 
git fetch
git status
Wat betekent de uitkomst?
  • up to date → Alles is gelijk. Je hoeft niets te doen.
  • ahead → Jouw computer loopt voor. Je hebt wijzigingen die nog niet op GitHub staan. Typ save in Terminal om ze te versturen.
  • ahead → Jouw computer loopt voor. Je hebt wijzigingen die nog niet op GitHub staan. Typ de 3 Git-commando's (zie stap 5) om ze te versturen.
  • behind → GitHub loopt voor. Iemand anders heeft iets gewijzigd. Typ git pull om die wijzigingen op te halen.
  • modified → Je hebt iets aangepast op je computer, maar het is nog niet opgeslagen in Git. Typ save in Terminal om het op te slaan.
  • modified → Je hebt iets aangepast op je computer, maar het is nog niet opgeslagen in Git. Typ de 3 Git-commando’s (zie stap 5) om het op te slaan.
