Morgen Academy – Git workflow

Deze pagina is bedoeld voor mensen die niet dagelijks met Git of Terminal werken. Daarom leggen we hieronder uit wat je moet doen, waar je het doet en wanneer je welke stap gebruikt.

1. Vooraf instellen 2. Bestaand project 3. Nieuw project 4. Ik begin vandaag 5. Ik ga opslaan 6. Ik zit vast

Eerst het idee in gewone taal

GitHub is de centrale plek waar de laatste versie van een project staat.

1. GitHub bevat de centrale versie van het project
↓
2. Jij haalt dat project naar je computer of zet het er voor het eerst op
↓
3. Je werkt in die projectmap met Claude, Codex, Conductor of Antigravity
↓
4. Daarna stuur je je wijzigingen weer terug naar GitHub

Wat bedoelen we met “project”?

Met “project” bedoelen we gewoon de map op je computer waar de bestanden in staan. Bijvoorbeeld een map met index.html, afbeeldingen, styles en scripts.

Wat is de hoofdregel?

Je gebruikt steeds dezelfde Git-projectmap, ook als je wisselt tussen Claude, Codex, Conductor of Antigravity. De tool mag wisselen. De map niet.

Belangrijk Git-commando’s zoals git pull, git status en git push werken alleen in een map die al aan Git gekoppeld is.

1. Vooraf instellen

Deze dingen hoef je maar één keer te doen. Daarna wordt de rest makkelijker.

1
Installeer Git op je computer via git-scm.com.
2
Maak in je Documenten een map aan met de naam Repos.

Daar zet je voortaan al je projecten in. Zo weet je altijd waar ze staan.
3
Werk voortaan vanuit die mapstructuur, zodat je niet hoeft te zoeken waar een project staat.

Save shortcut instellen

Dit is optioneel, maar handig. Dan hoef je later niet steeds drie losse Git-commando’s te typen.

Wat is hier het doel? Als je deze shortcut instelt, kun je later in Terminal gewoon save typen in plaats van drie losse regels.
1
Open de app Terminal.
2
Kopieer het eerste commando hieronder, plak het in Terminal en druk op Enter.
3
Kopieer daarna het tweede commando hieronder, plak het in Terminal en druk weer op Enter.
4
Kopieer daarna het derde commando hieronder, plak het in Terminal en druk weer op Enter.
Stap 1 (Mac) 
nano ~/.zshrc
Stap 2 (Mac) 
alias save="git add . && git commit -m 'update' && git push"
Stap 3 (Mac) 
source ~/.zshrc
Goed om te weten Hierna werkt het commando save in elke Terminal die je opnieuw opent.
1
Windows-gebruikers hoeven dit stukje voorlopig niet te doen.
2
Gebruik op Windows voorlopig gewoon de 3 losse Git-commando’s verderop op deze pagina.
Windows-gebruikers 
Gebruik op Windows voorlopig liever de 3 losse Git-commando’s hieronder.
Zonder save shortcut 
git add .
git commit -m "update"
git push

2. Bestaand project

Gebruik deze route als het project al op GitHub staat.

Wat betekent dit? Als het project al op GitHub staat, haal je eerst een kopie van dat project naar je computer. Dat heet clonen.

Waar haal ik de repo-URL vandaan?

1
Open het project op GitHub in je browser.
2
Klik op de groene knop Code.
3
Kopieer de HTTPS-link. Dat is de URL die je hieronder op de plek van REPO-URL zet.
1
Open Terminal (Mac) of PowerShell (Windows).
2
Typ het commando hieronder en druk op Enter.
Project ophalen van GitHub (Mac) 
cd ~/Documents/Repos
git clone REPO-URL projectnaam
Project ophalen van GitHub (Windows) 
cd %USERPROFILE%\Documents\Repos
git clone REPO-URL projectnaam
Wat moet je vervangen? Vervang REPO-URL door de link van GitHub. Vervang projectnaam door een logische mapnaam, bijvoorbeeld website of internal-workflow.

3. Nieuw project

Gebruik deze route als het project nog niet op GitHub staat.

Wat bedoelen we hier precies? Stel: je hebt al een map op je computer met bestanden erin, maar GitHub weet nog helemaal niet dat dit project bestaat. Dan moet je die map één keer voor het eerst op GitHub zetten.
1
Maak eerst een nieuwe lege repo op GitHub.
2
Open daarna de map op je computer waar de projectbestanden in staan.
3
Open Terminal of PowerShell in precies die map.
4
Kopieer daarna het blok hieronder, plak het in Terminal of PowerShell, vervang REPO-URL en druk op Enter.
Project voor het eerst naar GitHub sturen 
git init
git add .
git commit -m "first commit"
git remote add origin REPO-URL
git push -u origin main
Wat doet dit? Hiermee maak je van jouw bestaande map een Git-projectmap en stuur je die voor het eerst naar GitHub.

4. Ik begin vandaag

Gebruik deze stappen als je verder werkt aan een project dat al als Git-project bestaat.

Hoe kom ik in de juiste map?

1
Open Finder.
2
Ga naar je projectmap in Documenten → Repos.
3
Open de map van het project, bijvoorbeeld internal-workflow.
4
Rechtsklik in die map en kies Open Terminal at Folder.
5
Nu staat je Terminal meteen op de juiste plek. Dán typ je git pull.
1
Open Verkenner.
2
Ga naar Documenten → Repos.
3
Open de map van het project.
4
Klik in de adresbalk, typ powershell en druk op Enter.
5
PowerShell opent nu in de juiste map. Dán typ je git pull.
Haal de nieuwste versie op 
git pull

5. Ik ga opslaan

Je werkt meestal in Claude Code, Codex, Antigravity of Conductor. Het opslaan doe je daarna in Terminal of PowerShell, in dezelfde Git-projectmap.

Waar typ ik dit? Niet in Claude of Antigravity zelf, maar in Terminal of PowerShell, terwijl die open staat in dezelfde Git-projectmap waarin je net gewerkt hebt.
1
Werk aan je project in Claude, Codex, Antigravity of Conductor.
2
Open Terminal of PowerShell in diezelfde Git-projectmap.
3
Typ save óf de 3 losse Git-commando’s.
Korte versie met shortcut 
save
Zonder shortcut 
git add .
git commit -m "update"
git push

Twijfel je wat de laatste versie is?

Gebruik dit als je even niet meer weet of GitHub of jouw computer de nieuwste versie heeft.

Check laatste versie 
git fetch
git status
Wat betekent de uitkomst?
  • up to date → lokaal en GitHub zijn gelijk
  • ahead → jouw computer heeft nieuwere wijzigingen
  • behind → GitHub heeft nieuwere wijzigingen
  • modified → lokaal bestand aangepast maar nog niet opgeslagen in Git

6. Ik zit vast

Doe dan eerst dit. Meestal kom je hier al mee verder.

Check wat Git ziet 
git status
Check in welke projectmap je zit 
git rev-parse --show-toplevel
Haal nieuwste versie op 
git pull
Krijg je een fout als “not a git repository”? Dan zit je niet in een Git-projectmap. Ga dan terug naar de map die je van GitHub gecloned hebt, of zet het project eerst voor het eerst op GitHub als het nog nieuw is.
