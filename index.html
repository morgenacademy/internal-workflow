Morgen Academy – Git workflow

Stap voor stap uitleg hoe we projecten bijhouden met GitHub.

Eerst het idee

GitHub bevat de centrale versie van het project
↓
Je haalt het project naar je computer
↓
Je werkt lokaal met Claude / Codex / Antigravity
↓
Je stuurt wijzigingen weer naar GitHub
Wat is een project?
Een map met bestanden zoals index.html, afbeeldingen en scripts.

Project ophalen van GitHub

Als het project al op GitHub staat.

Mac 
cd ~/Repos
git clone REPO-URL projectnaam
Windows 
cd C:\Repos
git clone REPO-URL projectnaam

Project voor het eerst op GitHub zetten

Als je project nog niet op GitHub staat.

Eerste upload 
git init
git add .
git commit -m "first commit"
git remote add origin REPO-URL
git push -u origin main

Begin van je werkdag

1
Open de projectmap.
2
Open Terminal in die map.
3
Haal de nieuwste versie op.
Nieuwste versie ophalen 
git pull

Na je werk opslaan

Opslaan naar GitHub 
git add .
git commit -m "update"
git push

Twijfel je wat de laatste versie is?

Check status 
git fetch
git status
Wat betekent de uitkomst?
  • up to date → lokaal en GitHub zijn gelijk
  • ahead → jouw computer heeft nieuwere wijzigingen
  • behind → GitHub heeft nieuwere wijzigingen
  • modified → lokaal bestand aangepast maar nog niet opgeslagen
